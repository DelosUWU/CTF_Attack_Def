# PatchBoard

PatchBoard - учебный сервис для Attack-Defense CTF с намеренно оставленными уязвимостями. По легенде это доска приватных карточек для передачи задач внутри команды реагирования на инциденты: пользователь регистрируется, создает приватную карточку и позже получает ее через обычный авторизованный API.

Сервис написан без веб-фреймворков: backend использует стандартную библиотеку Python и SQLite, frontend сделан на обычных HTML, CSS и JavaScript. Чекер и PoC-эксплойты используют библиотеку `requests`.

## Запуск

```bash
docker pull python:3.12-slim
```

```bash
docker compose up --build
```

Сервис работает на порту `8080`:

```text
http://127.0.0.1:8080/
```

Данные сервиса хранятся в Docker volume `patchboard-data`, поэтому флаги не удаляются при обычном перезапуске контейнера.

## Бизнес-логика

1. Зарегистрировать пользователя: `POST /api/register`.
2. Войти в аккаунт: `POST /api/login`.
3. Создать приватную карточку: `POST /api/cards`.
4. Получить свою карточку: `GET /api/cards/<id>`.
5. При необходимости открыть карточку по receipt-ссылке: `GET /api/cards/<id>/receipt?code=<code>`.

Флаг хранится в теле приватной карточки. Чекер кладет флаг через регистрацию пользователя и создание карточки, а затем получает его обратно через авторизованный API карточек.

## Чекер

При необходимости установите зависимости чекера на машине, с которой он запускается:

```bash
python -m pip install -r requirements.txt
```

Поддерживаемые команды:

```bash
python checker.py 127.0.0.1 8080 check
python checker.py 127.0.0.1 8080 put
python checker.py 127.0.0.1 8080 get <flag_id> <flag>
python checker.py 127.0.0.1 8080 get_flags
```

Также поддерживается порядок аргументов, где команда указана первой:

```bash
python checker.py check 127.0.0.1 8080
```

Коды возврата:

```text
OK=101
CORRUPT=102
MUMBLE=103
DOWN=104
CHECKER_ERROR=110
```

Команда `put` печатает публичный `flag_id` в формате `username:card_id`. Если флаг не передан в `put`, чекер сам генерирует случайный флаг, соответствующий регулярному выражению `[A-Z0-9]{31}=`.

Команда `get_flags` выводит все флаги, которые были успешно сгенерированы и положены этим чекером.

## Уязвимости

Сервис специально сделан небезопасным для учебного CTF. Не используйте его как реальное приложение.

### 1. SQL Injection в поиске карточек

Эндпоинт: `GET /api/cards/search?q=...`

Параметр поиска вставляется в SQL-запрос через f-string. Авторизованный атакующий может внедрить условие, которое обходит фильтр по владельцу и возвращает preview чужих карточек.

PoC:

```bash
python exploits/search_sqli.py 127.0.0.1 8080
```

### 2. Предсказуемые receipt-коды

Эндпоинт: `GET /api/cards/<id>/receipt?code=...`

Receipt-код считается как `sha1(username + ":" + card_id)[:12]`. Публичный `flag_id`, который возвращает чекер, уже содержит `username:card_id`, поэтому атакующий может вычислить код и прочитать карточку без пароля владельца.

PoC:

```bash
python exploits/receipt_code.py 127.0.0.1 8080 <flag_id>
```

### 3. Path Traversal при скачивании документов

Эндпоинт: `GET /api/library?doc=...`

Обработчик библиотеки склеивает имя документа с директорией `service/library` без нормализации пути. Атакующий может запросить `../data/ledger.db`, скачать SQLite-базу сервиса и извлечь тела карточек.

PoC:

```bash
python exploits/library_traversal.py 127.0.0.1 8080
```

### 4. Просмотр каталогов через URL-индексацию в поиске

Эндпоинт: `GET /api/cards/search?q=...`

В обработчике поиска есть скрытая логика индексации URL. Если авторизованный пользователь введет в поисковую строку URL-подобное значение с `../`, например `https://patchboard/../../../`, сервис разрешит путь относительно `service/library` и вернет листинг каталога прямо в результатах поиска. Так атакующий может просматривать каталоги внутри контейнера сервиса.

PoC:

```bash
python exploits/search_directory_listing.py 127.0.0.1 8080
```

## Структура проекта

```text
service/app.py              backend сервиса
service/static/             frontend
service/library/            документы библиотеки
checker.py                  Attack-Defense чекер
exploits/                   PoC-скрипты для уязвимостей
Dockerfile                  Docker-образ сервиса
docker-compose.yml          конфигурация запуска
requirements.txt            зависимости чекера и PoC-скриптов
```
